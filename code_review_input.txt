I have updated DeviceConnectionManager.mm to support location simulation on iOS 17+ (CoreDevice).
Key changes:
- Added handles for CoreDeviceProxy, Adapter, RsdHandshake, RemoteServer, and LocationSimulation (new API).
- Updated cleanupInternal to free these handles.
- Modified simulateLocationWithLatitude:longitude: to attempt the CoreDevice connection sequence if no simulation is active.
- The sequence includes: core_device_proxy_connect -> get rsd port -> create tcp adapter -> adapter_connect -> rsd_handshake_new -> remote_server_connect_rsd -> location_simulation_new.
- Added fallback to legacy lockdown location simulation if CoreDevice fails.
- Updated clearSimulatedLocation to handle both old and new handles.

Please review the error handling, handle ownership (consumption), and logic flow.
